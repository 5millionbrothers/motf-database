# 운영 안정화 확인표

## 적용 전

- [ ] Supabase 프로젝트 ID가 운영 대상과 일치한다.
- [ ] DB 백업을 만들었다.
- [ ] 17번 적용 여부를 확인했다.
- [ ] 18번 SQL 전체를 검토했다.

## 이용자 권한

- [ ] 비로그인 상태에서 승인 업장과 활성 상품만 조회된다.
- [ ] 로그인한 일반 이용자도 같은 공개 상품을 조회할 수 있다.
- [ ] 일반 이용자가 다른 이용자의 예약·주문·채팅을 조회할 수 없다.
- [ ] 파트너 또는 정지 계정으로 예약·주문을 생성할 수 없다.

## 금액과 상품

- [ ] 브라우저 요청에서 상품 가격·이름을 바꿔도 DB 원본으로 저장된다.
- [ ] 다른 공판장의 상품 ID로 주문할 수 없다.
- [ ] 0 이하 수량과 비활성 상품 주문이 거절된다.
- [ ] 객실 최대 인원을 초과한 예약이 거절된다.
- [ ] 상품 설명을 수정해도 기존 상품 ID가 유지된다.

## 사장님·관리자

- [ ] 일반 이용자 계정이 사장님 대시보드로 이동하지 않는다.
- [ ] 사장님은 자기 업장의 거래와 채팅만 볼 수 있다.
- [ ] 관리자는 전체 문의 상태를 RPC로 변경할 수 있다.
- [ ] 공개된 데모 아이디와 비밀번호로 우회 로그인할 수 없다.

## 실패 시

오류 메시지, 실행한 SQL 번호, 계정 역할, 요청 시각을 기록하고 다음 배포를 중단합니다.
