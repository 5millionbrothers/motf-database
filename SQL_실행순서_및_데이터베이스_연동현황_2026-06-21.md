## 주의사항

- 이 폴더의 SQL은 **번호 순서대로 변화 내역을 보관하는 마이그레이션 기록**이다.
- 이미 실행한 SQL을 새 프로젝트가 아닌 현재 프로젝트에서 처음부터 다시 실행하지 않는다.
- 현재 프로젝트에는 01~16번이 실행되어 있으며, 17번은 이용자·사장님 v8 배포 전에 실행해야 한다.
- 02번과 05번은 당시 운영자 이메일을 지정하기 위한 일회성 SQL이다. 새 환경에서는 실제 운영자 이메일에 맞게 수정해야 한다.
- `service_role` 또는 비밀 키는 프론트엔드·GitHub에 절대 올리지 않는다. 현재 사용하는 Publishable Key는 공개 클라이언트용 키다.

## 현재 실제 연동이 확인된 기능

### 공통 계정

- Supabase 이메일 회원가입, 이메일 인증, 로그인, 자동 로그인 유지, 로그아웃
- 이용자·파트너·운영팀 역할 구분
- 이용자 학교·소속 및 연락처 저장
- 파트너 업장 정보 저장
- 운영팀의 가입 승인·거절, 계정 정지·재개

### 숙소·공판장 정보

- 파트너의 객실·상품명, 설명, 가격, 인원 정보 저장
- 업장 대표 사진과 객실·상품 사진 업로드
- 이용자 사이트의 실제 승인 업장·상품·사진 조회
- 운영팀의 상품 개수 확인 및 전체 상품 공개·숨김 처리

### 예약·주문

- 이용자의 실제 숙소 예약 요청 저장
- 이용자의 실제 공판장 주문 저장
- 사장님 화면의 예약·주문 확인, 확정, 거절
- 운영팀 화면의 전체 예약·주문 확인 및 대신 처리
- 이용자 마이페이지의 예약·주문 처리 상태 조회

### 채팅

- 이용자가 실제 업장을 선택해 대화 시작
- 이용자와 사장님의 메시지 송수신
- 운영팀의 업장별 전체 대화 읽기 전용 모니터링
- 실시간 메시지 갱신

## v8에 구현되었지만 아직 최종 적용 확인이 필요한 기능

- 사장님 답변을 이용자 채팅창 왼쪽에 표시
- 사장님 또는 이용자가 대화를 확인했을 때 `읽음` 상태 저장
- 이용자 고객센터의 실제 문의·분쟁 접수
- 이용자의 최근 문의 및 처리 상태 조회
- 운영팀의 실제 문의·분쟁 목록 조회와 `접수 → 처리 중 → 완료` 변경

적용 순서: **17번 SQL 실행 → 이용자 v8 배포 → 사장님 v8 배포 → 양쪽 테스트**

## 테이블은 있으나 화면까지 아직 실제 연결되지 않은 기능

- 이용자 리뷰 작성·조회와 운영팀 리뷰 숨김 관리
- 커뮤니티 글 작성·조회와 운영팀 게시물 숨김 관리
- 문의·분쟁에 특정 예약 또는 업장을 직접 연결하는 선택 기능
- 문의 처리 담당자 메모를 화면에서 작성·조회하는 기능
